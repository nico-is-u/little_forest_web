# 产品需求说明书（草稿）

## 1. 产品概述

- 产品名称（暂定）：小森邻
- 一句话介绍（这个产品是做什么的？）：
  - 小森邻是一个社区服务平台，为用户提供代取快递，代叫网约车，代买药品等基础服务，额外运营团购类的电商模块作为主要盈利来源。
- 主要平台：
  - [x] 微信小程序
  - [x] H5

## 2. 目标用户

- 目标用户群体：
  - 中老年群体，退休人员
- 用户的典型特征：
  - 年龄：50岁以上
  - 生活习惯：对电子设备不熟悉，对互联网不熟悉，对移动支付，网约车，跑腿服务等软件不熟悉。除去刚需消费外，习惯积蓄，有一定的消费能力。比起线上社交，更注重社区的群聚效应，邻里关系

## 3. 使用场景

- 场景 1：老人在家
  - 角色：老人
  - 场景描述：老人发现药物用完了急需药物
  - 想完成的事情：找人帮忙购买同款药物
- 场景 2：老人在家
  - 角色：老人
  - 场景描述：腿脚不方便下楼太麻烦
  - 想完成的事情：需要找人代拿快递或者重物件
- 场景 3： 在家想出门
  - 角色：老人
  - 场景描述：想出门公交车难等摩托车安全系数不高想打车
  - 想完成的事情：想找人帮忙叫车
- 场景 4： 参与社区团购
  - 角色：老人
  - 场景描述：在程序内见到类似拼多多的特卖推荐，详情页内有社区群友的购买记录
  - 想完成的事情：想参与购买

## 4. 核心功能列表

 - 代取快递
 - 代叫网约车
 - 代买药品
 - 参与社区团购

### 4.1 用户账号相关

- 登录方式：微信一键登录 / 手机号 + 验证码（根据宿主环境决定）
- 用户资料：昵称、头像、手机、收货地址

### 4.2 主要业务功能

- 功能 A：代取快递
  - 说明：用户填入取件码，快递员在快递柜或驿站取件后拍照确认快递件正确以及按快递件尺寸确认费用，用户确认后，快递员送货上门
  - 谁会用：腿脚不便，不擅长搬运大型快递件的老人
- 功能 B：代叫网约车
  - 说明：人工客服方式接听用户的请求，确认用户需求和网约车费以及服务费后，叫车到用户位置
  - 谁会用：不会用网约车软件的中老年用户
- 功能 C：代买药品
  - 说明：用户在程序内搜索需要的药品，查看药品详情，确认需求后，人工客服联系用户，确认用户需求和药品价格以及服务费后，用户付款，人工客服到用户位置取货，用户确认后，人工客服将药品送到用户位置
  - 谁会用：需要购买药品的中老年用户

### 4.3 管理/运营相关

- 需要有管理员后台，管理员可以管理用户账号，查看订单，处理异常情况
- 内容不需要审核
- 需要数据统计，用户数、订单数、营收等
- 智能客服系统，查看消息记录，处理用户问题

## 5. 页面结构（初版）

> 先列页面，不要求长什么样，只要知道“有哪几页，每一页大概做什么”。
- 注册页+登录页：
  - 在不影响登录表单的背景，展示温馨的使用场景演示
  - 微信一键登录（小程序环境）
  - 手机号 + 验证码（H5环境）
  - 允许暂时跳过
- 首页：
  - （核心功能）代取快递
    - 直接以input表单的形式作为二级页面的入口，用户填入信息后点击提交实际上进入代取快递的二级页面
  - （核心功能）代叫网约车
    - 直接以双input表单的形式上下展示，第一个表单自动抓取用户当前地理位置，第二个表单用户输入目的地，点击第二个表单进入二级页面
  - （核心功能）代买药品
    - 直接以select表单的形式提示老人遇到的身体问题，如头疼，感冒，发烧等，提供按钮进入二级页面
  - 团购的商品列表，列出推荐位的商品及价格
- 代取快递：
  - input表单，手动输入取件码或拍照自动获取
  - 快递件尺寸选择（默认小），radio表单
  - 确认取件按钮，点击后提交订单
- 代叫网约车：
  - 直接以双input表单的形式上下展示
  - 确认叫车按钮，点击后提交订单
- 代买药品
  - select表单，提示老人遇到的身体问题，如头疼，感冒，发烧等
  - 确认购买按钮，点击后提交订单
- 团购页面
  - 参考拼多多的团购页面，展示团购商品列表，商品详情，用户评价，团购时间等
- 我的页面
  - 个人信息页
    - 展示用户的个人信息，如头像、昵称、手机号等
    - 提供修改个人信息的功能
  - 收货地址页
    - 展示用户的收货地址列表
    - 提供添加、编辑、删除收货地址的功能
  - 显示用户订单列表
    - 展示用户的订单列表，包括订单号、商品信息、订单状态、订单金额等
    - 提供订单详情的查看功能
  - 显示用户收藏的商品
    - 展示用户收藏的商品列表，包括商品图片、商品名称、商品价格等
    - 提供取消收藏的功能
  - 显示用户的评价
    - 展示用户对商品的评价列表，包括评价内容、评价时间等
    - 提供发表评价的功能
  - 显示用户的团购记录
    - 展示用户参与的团购记录，包括团购商品、团购时间、团购金额等
    - 提供团购详情的查看功能

## 6. 非功能性需求

- 体验相关：
  - 由于是针对中老年用户，所以页面需要简洁、明了，避免复杂的操作流程，用户可以快速上手，无需过多的解释。
  - 尽量多的svg动画提示用户操作，如点击按钮后出现小动画提示用户操作成功
